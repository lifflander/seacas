28  /**
29  Generate a cube mesh of size 'num_x' by 'num_y' by 'num_z' elements.
30  By default, the mesh is generated on a single processor. If 'proc_count' is
31  greater than 1, then the mesh will be distributed over 'proc_count' processors
32  and this process will get the portion of the mesh for 'my_proc'.
33  The mesh will be decomposed along the 'Z' axis so 'num_z' must be greater than
34  or equal to 'proc_count' and for even distribution of the hexes 'num_z' mod 'proc_count'
35  should be zero.
36 
37  The mesh can optionally include shell elements along each face of the cube mesh.
38  These are specified via the 'add_shell_block' function.
39 
40  The mesh can optionally include nodesets/sidesets along each
41  face of the cube mesh. These are specified via the
42  'add_nodesets' and 'add_sidesets' functions.
43 
44  If the 'parameters' string constructor is used, the string
45  is parsed to determine the intervals in each direction and,
46  optionally, additional information. The form of the string
47  is "IxJxK" where I, J, and K are the number of intervals
48  in the X, Y, and Z directions respectively and the "x" are
49  literal 'x' characters. For example, the constructor
50  GeneratedMesh("10x12x14") will create the same mesh as
51  GeneratedMesh(10,12,14)
52 
53  Additional valid options are:
54  - help -- no argument, shows valid options
55  - show -- no argument, prints out a summary of the
56  GeneratedMesh() parameters. The output will look similar
57  to:
58  \code
59  "10x12x8|shell:xX|bbox:-10,-10,-10,10,10,10|nodeset:xyz|sideset:XYZ|show"
60 
61  Mesh Parameters:
62  Intervals: 10 by 12 by 8
63  X = 2 * (0..10) + -10 Range: -10 <= X <= 10
64  Y = 1.66667 * (0..12) + -10 Range: -10 <= Y <= 10
65  Z = 2.5 * (0..8) + -10 Range: -10 <= Z <= 10
66  Node Count (total) = 1287
67  Element Count (total) = 1152
68  Block Count = 3
69  NodeSet Count = 3
70  SideSet Count = 3
71  \endcode
72 
73  - tets -- no argument - specifies that each hex should be
74  split into 6 tetrahedral elements. Cannot currently be used with
75  shells or sidesets.
76 
77  - shell -- argument = xXyYzZ which specifies whether there is a shell
78  block at that location. 'x' is minimum x face, 'X' is maximum x face,
79  similarly for y and z. Note that the argument string is a single
80  multicharacter string. You can add multiple shell blocks to a face,
81  for example, shell:xxx would add three layered shell blocks on the
82  minimum x face. An error is output if a non xXyYzZ character is
83  found, but execution continues.
84 
85  - nodeset -- argument = xXyYzZ which specifies whether there is
86  a nodeset at that location. 'x' is minimum x face, 'X' is
87  maximum x face, similarly for y and z. Note that the argument
88  string is a single multicharacter string. You can add multiple
89  nodesets to a face, for example, nodeset:xxx would add three
90  nodesets on the minimum x face. An error is output if a non
91  xXyYzZ character is found, but execution continues.
92 
93  - sideset -- argument = xXyYzZ which specifies whether there is
94  a sideset at that location. 'x' is minimum x face, 'X' is
95  maximum x face, similarly for y and z. Note that the argument
96  string is a single multicharacter string. You can add multiple
97  sidesets to a face, for example, sideset:xxx would add three
98  sidesets on the minimum x face. An error is output if a non
99  xXyYzZ character is found, but execution continues. If there
100  is a shell block specified on that face, then the sideset will
101  be on the shell elements; else the sideset will be on the hex
102  elements.
103 
104  - zdecomp -- argument = n0, n1, n2, ..., n#proc-1 which are the number
105  of intervals in the z direction for each processor in a pallel run.
106  If this option is specified, then the total number of intervals in the
107  z direction is the sum of the n0, n1, ... An interval count must be
108  specified for each processor. If this option is not specified, then
109  the number of intervals on each processor in the z direction is
110  numZ/numProc with the extras added to the lower numbered processors.
111 
112  - scale -- argument = xs, ys, zs which are the scale factors in the x,
113  y, and z directions. All three must be specified if this option is
114  present.
115 
116  - offset -- argument = xoff, yoff, zoff which are the offsets in the
117  x, y, and z directions. All three must be specified if this option
118  is present.
119 
120  - bbox -- argument = xmin, ymin, zmin, xmax, ymax, zmax
121  which specify the lower left and upper right corners of
122  the bounding box for the generated mesh. This will
123  calculate the scale and offset which will fit the mesh in
124  the specified box. All calculations are based on the currently
125  active interval settings. If scale or offset or zdecomp
126  specified later in the option list, you may not get the
127  desired bounding box.
128 
129  - rotate -- argument = axis,angle,axis,angle,...
130  where axis is 'x', 'y', or 'z' and angle is the rotation angle in
131  degrees. Multiple rotations are cumulative. The composite rotation
132  matrix is applied at the time the coordinates are retrieved after
133  scaling and offset are applied.
134 
135  The unrotated coordinate of a node at grid location i,j,k is:
136  \code
137  x = x_scale * i + x_off,
138  y = z_scale * j + y_off,
139  z = z_scale * k + z_off,
140  \endcode
141 
142  The extent of the unrotated mesh will be:
143  \code
144  x_off <= x <= x_scale * numX + x_off
145  y_off <= y <= y_scale * numY + y_off
146  z_off <= z <= z_scale * numZ + z_off
147  \endcode
148 
149  If an unrecognized option is specified, an error message will be
150  output and execution will continue.
151 
152  An example of valid input is:
153  \code
154  "10x20x40|scale:1,0.5,0.25|offset:-5,-5,-5|shell:xX"
155  \endcode
156 
157  This would create a mesh with 10 intervals in x, 20 in y, 40 in z
158  The mesh would be centered on 0,0,0 with a range of 10 in each
159  direction. There would be a shell layer on the min and max
160  x faces.
161 
162  NOTE: All options are processed in the order they appear in
163  the parameters string (except rotate which is applied at the
164  time the coordinates are generated/retrieved)
165  */
