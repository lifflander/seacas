28  /**
29  Generate a cube mesh of size 'num_x' by 'num_y' by 'num_z' elements.
30  By default, the mesh is gen_struc on a single processor. If 'proc_count' is
31  greater than 1, then the mesh will be distributed over 'proc_count' processors
32  and this process will get the portion of the mesh for 'my_proc'.
33  The mesh will be decomposed along the 'Z' axis so 'num_z' must be greater than
34  or equal to 'proc_count' and for even distribution of the hexes 'num_z' mod 'proc_count'
35  should be zero.
36 
37  The mesh can optionally include sidesets along each
38  face of the cube mesh. These are specified via the
39  'add_sidesets' function.
40 
41  If the 'parameters' string constructor is used, the string
42  is parsed to determine the intervals in each direction and,
43  optionally, additional information. The form of the string
44  is "IxJxK" where I, J, and K are the number of intervals
45  in the X, Y, and Z directions respectively and the "x" are
46  literal 'x' characters. For example, the constructor
47  GeneratedMesh("10x12x14") will create the same mesh as
48  GeneratedMesh(10,12,14)
49 
50  Additional valid options are:
51  - help -- no argument, shows valid options
52  - show -- no argument, prints out a summary of the
53  GeneratedMesh() parameters. The output will look similar
54  to:
55  \code
56  "10x12x8|bbox:-10,-10,-10,10,10,10|sideset:XYZ|show"
57 
58  Mesh Parameters:
59  Intervals: 10 by 12 by 8
60  X = 2 * (0..10) + -10 Range: -10 <= X <= 10
61  Y = 1.66667 * (0..12) + -10 Range: -10 <= Y <= 10
62  Z = 2.5 * (0..8) + -10 Range: -10 <= Z <= 10
63  Node Count (total) = 1287
64  Element Count (total) = 1152
65  Block Count = 3
66  SideSet Count = 3
67  \endcode
68 
69  - sideset -- argument = xXyYzZ which specifies whether there is
70  a sideset at that location. 'x' is minimum x face, 'X' is
71  maximum x face, similarly for y and z. Note that the argument
72  string is a single multicharacter string. You can add multiple
73  sidesets to a face, for example, sideset:xxx would add three
74  sidesets on the minimum x face. An error is output if a non
75  xXyYzZ character is found, but execution continues.
76 
77  - zdecomp -- argument = n0, n1, n2, ..., n#proc-1 which are the number
78  of intervals in the z direction for each processor in a pallel run.
79  If this option is specified, then the total number of intervals in the
80  z direction is the sum of the n0, n1, ... An interval count must be
81  specified for each processor. If this option is not specified, then
82  the number of intervals on each processor in the z direction is
83  numZ/numProc with the extras added to the lower numbered processors.
84 
85  - scale -- argument = xs, ys, zs which are the scale factors in the x,
86  y, and z directions. All three must be specified if this option is
87  present.
88 
89  - offset -- argument = xoff, yoff, zoff which are the offsets in the
90  x, y, and z directions. All three must be specified if this option
91  is present.
92 
93  - bbox -- argument = xmin, ymin, zmin, xmax, ymax, zmax
94  which specify the lower left and upper right corners of
95  the bounding box for the gen_struc mesh. This will
96  calculate the scale and offset which will fit the mesh in
97  the specified box. All calculations are based on the currently
98  active interval settings. If scale or offset or zdecomp
99  specified later in the option list, you may not get the
100  desired bounding box.
101 
102  - rotate -- argument = axis,angle,axis,angle,...
103  where axis is 'x', 'y', or 'z' and angle is the rotation angle in
104  degrees. Multiple rotations are cumulative. The composite rotation
105  matrix is applied at the time the coordinates are retrieved after
106  scaling and offset are applied.
107 
108  The unrotated coordinate of a node at grid location i,j,k is:
109  \code
110  x = x_scale * i + x_off,
111  y = z_scale * j + y_off,
112  z = z_scale * k + z_off,
113  \endcode
114 
115  The extent of the unrotated mesh will be:
116  \code
117  x_off <= x <= x_scale * numX + x_off
118  y_off <= y <= y_scale * numY + y_off
119  z_off <= z <= z_scale * numZ + z_off
120  \endcode
121 
122  If an unrecognized option is specified, an error message will be
123  output and execution will continue.
124 
125  An example of valid input is:
126  \code
127  "10x20x40|scale:1,0.5,0.25|offset:-5,-5,-5"
128  \endcode
129 
130  This would create a mesh with 10 intervals in x, 20 in y, 40 in z
131  The mesh would be centered on 0,0,0 with a range of 10 in each
132  direction.
133 
134  NOTE: All options are processed in the order they appear in
135  the parameters string (except rotate which is applied at the
136  time the coordinates are gen_struc/retrieved)
137  */
